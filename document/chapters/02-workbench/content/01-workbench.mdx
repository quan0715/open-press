## 專案初始化

OpenPress 的第一步是建立一個文件 workspace。Workspace 裡會包含文件 source、章節、theme、components、media、設定檔與輸出目錄。你不是在產生一個單一檔案，而是在建立一個可以讓 AI 和人類一起工作的文件專案。

### 先讓 Agent 認識 OpenPress

若你使用 Claude Code、Cursor、Codex 或其他支援 skills 的 agent，可以先安裝 OpenPress skills：

```bash
npx skills add quan0715/open-press
```

安裝後，agent 會知道 OpenPress 的基本工作流：如何初始化 workspace、如何寫固定版面文件、如何調整 design tokens、如何驗證輸出，以及部署前需要確認使用者意圖。

### 建立 workspace

如果你已經知道要用哪個 style pack，可以直接初始化：

```bash
npx @open-press/cli init my-doc --pack editorial-monograph
```

`editorial-monograph` 適合比較正式的長文件；`claude-document` 則適合溫暖、工作筆記感的文件。未來 OpenPress 可以增加更多 style pack，讓不同文件類型有不同的起手風格。

### Init Prompt 範例

你也可以不要先想 command，而是直接請 agent 協助初始化：

```text
Using OpenPress, help me set up a new document workspace.

Audience: product users and technical evaluators.
Document type: public user guide.
Language: Traditional Chinese with English technical terms when useful.
Style: clear, calm, fixed-layout A4 document.

Ask me for the title and style pack if needed, then initialize the workspace.
After init, explain which files I should edit first.
```

好的 init prompt 會交代四件事：讀者、文件類型、語言、風格。這些資訊會影響 agent 如何拆章節、如何寫摘要、是否需要圖表，以及要選哪個 style pack。

### 初始化後會看到什麼

一個 OpenPress workspace 通常會長這樣：

| 路徑 | 用途 |
| --- | --- |
| `openpress.config.mjs` | 文件標題、輸出目錄、PDF、deploy 設定 |
| `document/index.tsx` | `config`、`sources`，以及 default-exported Press tree |
| `document/chapters/` | starter 預設的 manuscript MDX 來源；實際來源由 `mdxSource()` 註冊 |
| `document/theme/` | 版面、字體、顏色、頁面類型 |
| `document/components/` | 可重用圖表、視覺元件或特殊區塊 |
| `document/media/` | 圖片、附件與素材 |

初始化完成後，不需要急著寫完整本。先確認封面、目錄方向與第一章，再讓 agent 一章一章擴充。OpenPress 的價值來自可持續的工作流，而不是一次把所有內容塞進 prompt。
