## 版面、風格與可讀性

正式文件需要有穩定的視覺語言。OpenPress 用 style pack、theme tokens、page surfaces 和 components 管理這件事，讓你可以在不重寫整份文件的情況下調整風格。

### Style Pack 是起手方向

Style pack 決定文件初始的外觀與節奏。它通常包含封面、目錄、章節頁、正文排版、表格、圖表、caption 與 PDF-safe rules。

<TableCaption>內建 style pack 的起手方向</TableCaption>

| Style pack | 適合 |
| --- | --- |
| `editorial-monograph` | 白皮書、提案、正式長文 |
| `claude-document` | 工作筆記、教學講義、溫暖的說明文件 |

你可以把 style pack 想成「文件設計系統的起點」。它不是不可修改的模板，而是一組讓 agent 和作者有共同語言的版面規則。

### Theme Tokens

OpenPress theme 會把常見設計決策集中在 tokens 裡，例如：

<TableCaption>Theme token 控制的文件設計層面</TableCaption>

| Token 類型 | 控制什麼 |
| --- | --- |
| page size | A4、Letter、16:9 等固定格式 |
| margin | 內容區與頁面邊界 |
| type scale | 標題、正文、caption、code |
| color | ink、paper、accent、muted |
| rhythm | paragraph、table、figure 間距 |

當你想調整風格時，優先讓 agent 改 tokens 或既有 theme layer，而不是在每個章節裡塞 inline style。這樣才能維持整份文件一致。

### Components 適合放什麼

內容裡若出現需要重複使用的視覺元素，可以抽成 component：

<TableCaption>OpenPress component 適合承載的內容類型</TableCaption>

| 元件 | 適合情境 |
| --- | --- |
| 比較表 | 重複出現的產品比較、流程比較 |
| 圖解 | 架構、資料流、文件流程 |
| callout | 注意事項、限制、決策 |
| specimen | 字體、色票、style preview |

元件的價值不是炫技，而是讓複雜內容有一致的表達方式。當文件越長，元件越能避免每一頁都長得像臨時拼起來。

### 內容層級比裝飾重要

固定版面文件最常見的問題不是不夠漂亮，而是層級不清楚。OpenPress 建議先處理這些問題：

<TableCaption>固定版面文件常見問題與調整方式</TableCaption>

| 問題 | 調整方式 |
| --- | --- |
| 每段都很長 | 拆成短段，讓每頁更容易掃讀 |
| 標題太多 | 合併小標，避免目錄變碎 |
| 表格太密 | 拆表或改成多個小節 |
| 圖沒有說明 | 補 caption，讓讀者知道該看什麼 |
| 內容塞不下 | 拆章節，不要壓縮字級硬塞 |

OpenPress 可以讓 agent 幫你調風格，但最重要的設計決策仍然是：這份文件要讓讀者用什麼順序理解事情。
