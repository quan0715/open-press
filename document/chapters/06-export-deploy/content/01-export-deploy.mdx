## Showcase 與公開範例

OpenPress 文件可以各自部署到 Cloudflare Pages，再由官網 showcase 連結過去。官網不需要複製每份文件內容，只要維護一份 registry，讓使用者看到不同文件類型的公開成果。

### 為什麼需要 Showcase

Showcase 的任務很直接：讓第一次接觸 OpenPress 的人看到「它實際能做出什麼」。比起列功能，公開文件更能說明 OpenPress 的價值。

<TableCaption>Showcase 中每個視覺線索代表的意義</TableCaption>

| Showcase 看到的東西 | 代表什麼 |
| --- | --- |
| 封面 | style pack 與文件身份 |
| 長型卡片 | OpenPress 偏向正式文件，而不是一般網站縮圖 |
| Public URL | 每份文件都能獨立部署 |
| 多種主題 | OpenPress 不只服務單一使用案例 |

### 目前的公開範例

<TableCaption>目前官網 showcase 收錄的公開文件</TableCaption>

| 文件 | 說明 | URL |
| --- | --- | --- |
| OpenPress User Story Book | OpenPress 自己的公開使用者指南 | `https://open-press-story.pages.dev` |
| 資料結構筆記 | Claude-like 教學筆記風格的資料結構文件 | `https://data-structure-note.pages.dev/#page-01` |

這些範例不需要塞進同一個 Pages project。每份文件可以有自己的部署、自己的 theme、自己的 public URL。Showcase 只負責索引和導流。

### Gallery 卡片應該簡單

第一版 showcase 不需要過多 metadata。對使用者來說，最重要的是封面、標題、簡短說明與連結。

<TableCaption>Showcase 第一版保留與延後的欄位</TableCaption>

| 保留 | 暫時不做 |
| --- | --- |
| cover | filter/search |
| title | validation matrix |
| description | 過多 tag |
| document link | runtime screenshot fetch |

等公開範例變多，再考慮分類、搜尋或自動抓取遠端 manifest。第一版的目標是讓作品先被看見，而不是建立複雜 registry 系統。

### 用公開範例反推自己的文件

當你要做新文件時，可以先看 showcase 裡哪一種比較接近：

<TableCaption>依文件目的選擇可參考的公開範例</TableCaption>

| 你要做的文件 | 可以參考 |
| --- | --- |
| 產品指南 | OpenPress User Story Book |
| 教學筆記 | 資料結構筆記 |
| 商業提案 | `editorial-monograph` starter |
| 工作說明 | `claude-document` starter |

OpenPress 的設計是讓每份文件都能逐步變成公開成果。Showcase 則是把這些成果收在一起，讓使用者快速理解可能性。
