## 開源貢獻者附錄

前面的章節面向 OpenPress 使用者；本章才集中說明 framework 開發者與開源貢獻者需要知道的邊界。OpenPress 本身也用這份文件 dogfood 自己，所以 contributor 修改 framework 時，應該用這份文件驗證真實輸出。

### Framework repo 的幾個區域

<TableCaption>OpenPress framework repo 的主要區域與責任</TableCaption>

| 區域 | 責任 |
| --- | --- |
| `packages/core` | engine、React runtime、Workbench、CLI commands |
| `packages/cli` | npm CLI 包裝與 init 入口 |
| `skills/` | agent skills 與 starter documents |
| `apps/web` | open-press.dev 官網 |
| root `document/` | 這份公開 user guide / dogfood 文件 |

Contributor 不應把 framework bug 修在 root `document/` 裡，也不應把這份文件內容打包進 `@open-press/core`。文件 source 可以被 repo 追蹤，但 npm package 邊界由 package `files` 欄位決定。

### 兩條 Dev Loop

OpenPress framework 需要兩個本機入口：

<TableCaption>Framework 開發時的兩條本機 dev loop</TableCaption>

| Command | 用途 |
| --- | --- |
| `pnpm run dev:workspace` | 開 root dogfood document，檢查 document pipeline |
| `pnpm run dev:web` | 開官網，檢查 landing/showcase/docs experience |

預設 `pnpm run dev` 應該指向 workspace，因為 framework 主要風險通常在文件 pipeline：pagination、reader、PDF、source mapping、deploy stage。

### Root document 和 starter document

<TableCaption>Root dogfood document 與 style starter document 的責任差異</TableCaption>

| 文件 | 位置 | 責任 |
| --- | --- | --- |
| root dogfood | `document/` | 驗證真實內容、PDF、reader、deploy |
| style starter | `skills/<pack>/starter/document` | 驗證新 workspace 起手和風格哲學 |

Style pack starter 可以自由覆蓋 theme，因為它的任務就是定義風格。但它仍要保留 OpenPress workspace contract，讓 CLI init、Workbench、export 和 PDF 都能正常運作。

### Contributor 變更後要跑什麼

最小驗證不應只有 typecheck。OpenPress 的核心價值是文件交付，所以要回到實際產出：

```bash
pnpm run build:doc        # validates + renders dogfood document
pnpm run openpress:pdf
```

如果改的是官網，也要跑：

```bash
pnpm --filter web typecheck
pnpm --filter web build
```

如果改的是 init、starter 或 package 邊界，還應該另外檢查 CLI init 出來的新 workspace，以及 package tarball 是否沒有混入 root `document/`。

### 部署與 custom domain

這份文件的 Pages project 是 `open-press-story`。官網 project 和 story project 應分開，避免 OpenPress 官網與 dogfood 文件互相污染部署責任。

`story.open-press.dev` 是 custom domain 設定，不應藏在每次 deploy command 裡。Deploy pipeline 負責把文件發布到 Pages；DNS 和 custom domain 是 Cloudflare zone 層級的一次性設定。

### Contributor 回報格式

完成變更時，回報應至少包含：

<TableCaption>Contributor 完成變更時應回報的資訊</TableCaption>

| 項目 | 例子 |
| --- | --- |
| Source 變更 | `packages/core/engine/...`、`apps/web/...`、`document/...` |
| 驗證 | 實際跑過的 command 與結果 |
| 產出 | reader URL、PDF path、deploy URL |
| 風險 | 尚未部署、custom domain 未生效、未跑 full test |
