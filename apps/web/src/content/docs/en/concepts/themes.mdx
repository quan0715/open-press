In OpenPress, the design of themes returns to the essence of web styling: pure CSS and Design Tokens. We deliberately avoid complex JavaScript theme runtimes or massive configuration objects.

## Why is a theme just CSS?

Many modern frameworks lean toward encapsulating styling logic in JavaScript, but OpenPress chooses a different path:

1. **The Single Source of Truth:** By using pure CSS variables (Tokens) as the foundation for all colors, fonts, and spacing, we ensure consistency of visual settings across the entire document. Any component, whether built-in or custom, can seamlessly access these native styling primitives.
2. **Deep Integration with Page Geometry:** The design of physical documents (like an A4 report or a presentation) heavily relies on precise physical dimensions. At build time, the OpenPress engine dynamically injects the page width, height, and proportions as CSS variables. This makes themes aware of the physical space they inhabit, thereby implementing responsive or adaptive typography.
3. **Clear Separation of Concerns:** Themes focus on "how it looks" (visual design), while React components (like covers or tables of contents) focus on "what the structure is" (markup language). This separation makes swapping designs extremely lightweight.

## Core Design Concepts

To understand OpenPress themes, you need to master several key abstraction layers:

### Design Tokens

Tokens are the atoms that make up a theme. They do not contain any typographic logic, but are merely "names" for specific color values or font families. For example, instead of hardcoding a specific blue hex code everywhere, you define an "accent color" token. This makes a comprehensive brand update require changes in just one place.

### Page Contract and Geometry

For documents that need to be exported to physical media (like PDF), the theme must define a "page contract." This involves defining the printable physical area, margins, and ensuring content does not overflow boundaries. The geometry variables injected by the engine play a critical role here, allowing a single set of styles to intelligently adapt to differently sized papers.

### Typographic Scales and Base Rules

This section handles the most common content elements in a document (like headings, paragraphs, lists). It establishes the fundamental rhythm of the document, ensuring that regardless of who writes the content, it presents consistent typographic proportions and a good reading experience.

### Evolution and Compatibility

Although OpenPress provides powerful mechanisms for developers to build custom page UIs through React and Tailwind, the theme system retains compatibility with legacy styling patterns. This design ensures that existing documents can migrate smoothly, while opening the door for more modern, component-driven design approaches.
