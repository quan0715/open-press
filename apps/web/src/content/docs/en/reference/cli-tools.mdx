Level 3 CLI tools support workspace operations, structural inspections, and state updates. These tools are independent of the daily build lifecycle.

## Source Code Operations

<ApiEntry
  name="search"
  kind="command"
  importFrom="open-press search . <query> [--json]"
  summary="Executes full-text search across MDX sources. Returns file paths, line numbers, columns, and preview strings of matches."
>
  <PropsTable
    title="Flags"
    rows={[
      { name: "<query>", type: "string", required: true, description: "Search target. Supports plain text or regular expressions." },
      { name: "--json", type: "flag", description: "Output in machine-readable JSON format." },
      { name: "--scope", type: "string", default: '"content"', description: "Specifies search scope. `content` (MDX body only) or `all` (includes frontmatter and metadata)." },
      { name: "--case-sensitive", type: "flag", description: "Forces case-sensitive search. Case-insensitive by default." },
    ]}
  />

  ### Example: JSON Search

  ```bash
  open-press search . "Figure" --json | jq '.matches[] | {file: .path, line, preview}'
  ```
</ApiEntry>

<ApiEntry
  name="replace"
  kind="command"
  importFrom="open-press replace . <from> <to> [--apply]"
  summary="Executes batch string replacement operations across MDX sources. Default mode is read-only preview."
>
  <PropsTable
    title="Flags"
    rows={[
      { name: "<from>", type: "string", required: true, description: "Target string or regular expression to search for." },
      { name: "<to>", type: "string", required: true, description: "Replacement string." },
      { name: "--apply", type: "flag", description: "Applies changes and writes to files. If not provided, only a preview is output." },
      { name: "--scope", type: "string", default: '"content"', description: "Same scope setting as the `search` command." },
      { name: "--include-code", type: "flag", description: "Expands replacement scope to fenced code blocks. Skipped by default to prevent accidental code changes." },
      { name: "--case-sensitive", type: "flag", description: "Forces case-sensitive search." },
      { name: "--json", type: "flag", description: "Output in machine-readable JSON format." },
    ]}
  />

  ### Example: Execute Replacement

  ```bash
  open-press replace . "old phrase" "new phrase" --apply
  ```
</ApiEntry>

## Environment and State Management

<ApiEntry
  name="doctor"
  kind="command"
  importFrom="open-press doctor ."
  summary="Reports workspace dependencies and package version status. Checks framework version and agent skills. Results cached for 24 hours."
>
  <PropsTable
    title="Flags"
    rows={[
      { name: "--json", type: "flag", description: "Output in machine-readable JSON format." },
      { name: "--no-cache", type: "flag", description: "Bypasses local cache to query the latest information directly from npm." },
    ]}
  />

  The `doctor` command exit code is always `0`. Automation flows should parse `report.stale` or `report.coreUpdateAvailable` from the JSON output to determine execution logic.
</ApiEntry>

<ApiEntry
  name="upgrade"
  kind="command"
  importFrom="open-press upgrade ."
  summary="Upgrades workspace dependencies and agent skills. This command modifies project files."
>
  Use with `doctor`; ensure a Git version control snapshot is created before execution.
</ApiEntry>
