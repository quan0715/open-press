`@open-press/cli` provides a three-level command-line interface, responsible for managing workspace bootstrapping, build processes, and project maintenance.

<div class="callout">
  <strong>Interface Stability Guarantee:</strong> Level 1 and Level 2 commands fall under the 1.0 stable contract specification. Level 3 tools are exclusively for use by the system, agents, and the workbench.
</div>

## CLI Level Architecture

<div class="cli-grid">
  <a class="cli-card" href="/docs/concepts/cli-lifecycle">
    <p class="cli-card__eyebrow">Level 1</p>
    ### Lifecycle
    <p>Covers daily build operations. Includes: <code>create</code>, <code>dev</code>, <code>build</code>, <code>preview</code>, <code>typecheck</code>.</p>
  </a>

  <a class="cli-card" href="/docs/reference/cli-outputs">
    <p class="cli-card__eyebrow">Level 2</p>
    ### Output targets
    <p>Manages the generation of deliverables beyond standard HTML. Includes: <code>openpress:pdf</code>, <code>openpress:image</code>, and <code>openpress:deploy</code>.</p>
  </a>

  <a class="cli-card" href="/docs/reference/cli-tools">
    <p class="cli-card__eyebrow">Level 3</p>
    ### Tools
    <p>System maintenance and agent tools. Includes: <code>search</code>, <code>replace</code>, <code>inspect</code>, <code>doctor</code>, <code>upgrade</code>, and <code>skills:sync</code>.</p>
  </a>
</div>

## API Reference Format

Each CLI command in this manual is documented using an API entry format:

- **Basic Information**: Includes command name, kind badge, invocation syntax, and a one-line summary.
- **Parameter List**: Itemizes all supported flags and properties.
- **Examples**: Standard usage scenario demonstrations.

Invocation syntax will contextually show the native `open-press` command or the `npm run` wrapper alias.
