<p>
    Tier 1 は日常的な Workspace ループをカバーしています。作成し、その中で開発し、出荷し、検証します。
    以下の各コマンドには、足場が構築された <code>package.json</code> に対応する <code>npm run</code> スクリプトがあり、
    これがほとんどの作成者や CI スクリプトが使用するインターフェースです。
  </p>

    <h3>Example: Create with title</h3>

```bash
npm create @open-press my-deck -- \
  --type slides \
  --title "Transport models"
```

    <h3>Example: Create into existing repo</h3>

```bash
# 既存の git リポジトリ内で、自動コミットなし、依存関係を自身で管理：
npm create @open-press ./docs -- \
  --type slides \
  --no-git \
  --no-install
pnpm install
```

    <h3>Example: Add another slides Press</h3>

```bash
open-press create appendix --type slides --title "Appendix"
```

    <h3>Example: Resulting file tree</h3>

```text
my-deck/
  package.json
  .gitignore
  press/
    my-deck/
      press.tsx          # 順序付きインデックス
      slides/
        intro/
          slide.tsx      # メタデータ + デフォルトコンポーネント
      themes/
        default.css
```

    <p>
      ドメイン固有の starter は CLI にはバンドルされていません。ドメイン skill をインストールし
      (<code>npx skills add &lt;owner/repo&gt;</code>)、エージェントにその skill の starter から
      <code>press/</code> を生成するように依頼してください。<code>slide.tsx</code> と <code>SlideMeta</code> の完全なコントラクトについては <a href="/docs/concepts/slides">Slides architecture</a> を参照してください。
    </p>
  </ApiEntry>

  <ApiEntry
    name="dev"
    kind="command"
    importFrom="npm run dev"
    summary="http://127.0.0.1:5173 でローカルのワークベンチを起動します。CSS、テーマトークン、および React UI クロムをホットリロードします。MDX コンテンツの編集は、エンジンのソースウォッチャーを通じて更新されます。"
  >
    <h3>Example: Start the workbench</h3>

```bash
npm run dev
# → workspace at http://127.0.0.1:5173/workspace
# → document preview at http://127.0.0.1:5173/`<press-slug>`/preview
```

    <p>
      ワークベンチはパスベースのルートを使用します。プロジェクトギャラリーには <code>/workspace</code>、
      インスペクター、ソース編集エンドポイント、コメントマーカーが有効になった特定のドキュメントプレビューには
      <code>/&lt;press-slug&gt;/preview</code> を使用します。
    </p>
  </ApiEntry>

  <ApiEntry
    name="build"
    kind="command"
    importFrom="npm run build"
    summary="dist-react/ にデプロイ可能なバンドルを生成します。ソースの検証、MDX → React エクスポート、および Vite プロダクションビルドを連鎖的に実行します。検証が失敗した場合、Vite が実行される前にビルドが中断されます。"
  >
    <h3>Example: Build for production</h3>

```bash
npm run build
# → dist-react/ ready to host
```

    <p>
      中間のエンジンステップ (<code>export</code>、<code>validate</code>) は高度な呼び出し元のために存在しますが、ユーザー向けではありません。これらは <code>build</code> 内で自動的に実行されます。レンダリングせずに検証するには、直接 <code>open-press validate .</code> を呼び出します。これは主に CI リントまたはエージェントのプリフライト用です。
    </p>
  </ApiEntry>

  <ApiEntry
    name="preview"
    kind="command"
    importFrom="npm run preview"
    summary="ワークベンチのクロムなしで、dist-react/ を静的サイトとして提供します。デプロイ前に公開されるリーダーが期待通りであることを確認するために使用します。"
  >
    <h3>Example: Preview after build</h3>

```bash
npm run build
npm run preview
# → reader at http://127.0.0.1:4173 (no workbench)
```
