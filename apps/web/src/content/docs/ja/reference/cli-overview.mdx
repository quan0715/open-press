`@open-press/cli` は、ワークスペースの起動、ビルドプロセス、およびプロジェクトのメンテナンスを管理するための3層のコマンドラインインターフェースを提供します。

<div class="callout">
  <strong>インターフェースの安定性保証：</strong> 第1層と第2層のコマンドは 1.0 安定契約の仕様に属します。第3層のツールは、システム、エージェント、およびワークベンチ専用です。
</div>

## CLI 階層アーキテクチャ

<div class="cli-grid">
  <a class="cli-card" href="/docs/concepts/cli-lifecycle">
    <p class="cli-card__eyebrow">第1層</p>
    ### ライフサイクル (Lifecycle)
    <p>日々のビルド操作をカバーします。含まれるもの：<code>create</code>、<code>dev</code>、<code>build</code>、<code>preview</code>、<code>typecheck</code>。</p>
  </a>

  <a class="cli-card" href="/docs/reference/cli-outputs">
    <p class="cli-card__eyebrow">第2層</p>
    ### 出力ターゲット (Output targets)
    <p>標準HTML以外の成果物の出力を管理します。含まれるもの：<code>openpress:pdf</code>、<code>openpress:image</code>、および <code>openpress:deploy</code>。</p>
  </a>

  <a class="cli-card" href="/docs/reference/cli-tools">
    <p class="cli-card__eyebrow">第3層</p>
    ### ツール (Tools)
    <p>システムのメンテナンスとエージェントツール。含まれるもの：<code>search</code>、<code>replace</code>、<code>inspect</code>、<code>doctor</code>、<code>upgrade</code>、および <code>skills:sync</code>。</p>
  </a>
</div>

## API リファレンスフォーマット

本マニュアルの各 CLI コマンドは、API エントリ形式で記録されています：

- **基本情報**：コマンド名、種類のラベル、呼び出し構文、および1行の概要が含まれます。
- **パラメータリスト**：サポートされているすべてのフラグ (Flags) とプロパティをリストします。
- **例**：標準的な使用シナリオのデモンストレーション。

呼び出し構文は、コンテキストに応じてネイティブの `open-press` コマンドまたは `npm run` ラッパーエイリアスを表示します。
