  <h2>ページのジオメトリ</h2>

  <p>
    <code>&lt;Frame&gt;</code>コントラクトには<code>page</code>プロップは含まれていません。紙またはキャンバスのサイズは、各<code>&lt;Press page&gt;</code>のJSXプロップで設定されます。1つのPressは1つの固定ジオメトリになります。混合ジオメトリのプロジェクトでは、Pressごとに1つのジオメトリを持つマルチPressの<code>&lt;Workspace&gt;</code>を使用します。
  </p>

  <ApiEntry

  >
    <p>
      <code>consumeArea(chainId)</code>は呼び出しごとにチェーンごとのカウンターをインクリメントするため、1つのFrame内で同じ<code>chainId</code>を持つ複数の<code>&lt;MdxArea&gt;</code>が、ソースの順序で別々の割り当てスロットにマッピングされます。
    </p>

  <h2>ロールの命名規則</h2>

  <p>
    <code>role</code>は自由な文字列ですが、テーマとドキュメントの整合性を保つために、フレームワークでは2セグメントの規則を文書化しています：
  </p>

  <ul>
    <li>
      <strong><code>document.&#42;</code></strong> — アロケーターを通じてMDXコンテンツを流し込む長文ページ：<code>document.cover</code>、<code>document.toc</code>、<code>document.content</code>、<code>document.back-cover</code>。
    </li>
    <li>
      <strong><code>canvas.&#42;</code></strong> — 1つのデザインされた表面と<code>overflow="truncate"</code>のMDXエリアを持つ固定フォーマットのページ：<code>canvas.slide</code>、<code>canvas.post</code>、<code>canvas.card</code>。
    </li>
    <li>
      <strong><code>manuscript.&#42;</code></strong> — バンドルされた原稿ヘルパーによって使用されます（<code>DefaultSectionPage</code>からの<code>manuscript.content</code>）。
    </li>
  </ul>
