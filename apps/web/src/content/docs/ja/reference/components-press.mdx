    <h3>Example: Single-document project</h3>

```tsx
// press/report/press.tsx
import { Press, Frame } from "@open-press/core";
import { mdxSource } from "@open-press/core/mdx";
import { Sections, Toc } from "@open-press/core/manuscript";

export default function ReportPress() {
  return (
    <Press
      slug="report"
      title="Transport models in dense networks"
      page="a4"
      componentsDir="./components"
      mediaDir="./media"
      sources={[
        mdxSource({ id: "story", preset: "section-folders", root: "report/chapters" }),
      ]}
    >
      <Frame frameKey="cover" role="document.cover">
        <Cover />          {/* subtitle / organization / author all live here */}
      </Frame>
      <Toc source="story" maxLevel={2} />
      <Sections source="story" />
      <Frame frameKey="back-cover" role="document.back-cover">
        <BackCover />
      </Frame>
    </Press>
  );
}
```

  <h2>ランタイムコントラクト</h2>

  <p>
    ページネーションの状態を<code>&lt;Press&gt;</code>に渡すことはありません。エクスポート中、OpenPressはツリーを複数回レンダリングします。最初はFrameとコンテンツスロットを検出するため、次に各ソースブロックが属する場所がわかった後にもう一度レンダリングします。そのランタイム状態はレンダラーの内部にあります。

  <h2>オーサリングルール</h2>

  <ul>
    <li>すべてのプロジェクトのルートは<code>&lt;Workspace&gt;</code>です。<code>&lt;Press&gt;</code>は常に子要素であり、トップレベルのコンポーネントになることはありません。</li>
    <li>1つのWorkspaceは1つまたはN個のPressの子要素を保持できます。単一ドキュメントのプロジェクトには1つあり、複数ドキュメントのプロジェクト（提案+ピッチ+ソーシャル）には複数あります。</li>
    <li>メタデータは<code>&lt;Press&gt;</code>のプロップに存在します — 1.0のコントラクトでは並行する設定ファイルはありません。</li>
    <li>タイトル以外の表示用コピー（サブタイトル、組織、作成者の署名、バージョン文字列）は、Pressのプロップではなく独自のCover JSXに入れます。</li>
    <li>Coverとバックカバーは、特別なAPIではなく、コンテンツヘルパーの前後に配置される通常の<code>&lt;Frame&gt;</code>コンポーネントです。</li>
    <li>レンダリング中にファイルをフェッチしたり、グローバルを変異させたり、ランダムな値に依存したりしないでください。レンダラーはツリーを複数回実行する場合があります。</li>
  </ul>
