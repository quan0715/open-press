第3層の CLI ツールは、ワークスペースの操作、構造検査、およびステータスの更新をサポートします。これらのツールは、日々のビルドのライフサイクルとは独立しています。

## ソースコード操作

<ApiEntry
  name="search"
  kind="command"
  importFrom="open-press search . <query> [--json]"
  summary="MDX ソースに対してフルテキスト検索を実行します。一致するアイテムのファイルパス、行番号、列、およびプレビュー文字列を返します。"
>
  <PropsTable
    title="パラメータ (Flags)"
    rows={[
      { name: "<query>", type: "string", required: true, description: "検索ターゲット。プレーンテキストまたは正規表現をサポートします。" },
      { name: "--json", type: "flag", description: "機械可読な JSON 形式で出力します。" },
      { name: "--scope", type: "string", default: '"content"', description: "検索範囲を指定します。`content` (MDX 本文のみ) または `all` (フロントマターとメタデータを含む)。" },
      { name: "--case-sensitive", type: "flag", description: "大文字と小文字を区別します。デフォルトでは区別しません。" },
    ]}
  />

  ### 例：JSON 検索

  ```bash
  open-press search . "Figure" --json | jq '.matches[] | {file: .path, line, preview}'
  ```
</ApiEntry>

  summary="MDX ソースに対して一括文字列置換操作を実行します。デフォルトモードは読み取り専用のプレビューです。"
>
  <PropsTable
    title="パラメータ (Flags)"
    rows={[
      { name: "<from>", type: "string", required: true, description: "検索ターゲット文字列または正規表現。" },

      { name: "--json", type: "flag", description: "機械可読な JSON レポートを出力します。" },

  />

  ### 例：置換の実行


## 環境と状態の管理

<ApiEntry
  name="doctor"
  kind="command"
  importFrom="open-press doctor ."
  summary="ワークスペースの依存関係とパッケージのバージョン状態を報告します。フレームワークのバージョンとエージェントスキルを確認します。結果は24時間キャッシュされます。"
>
  <PropsTable
    title="パラメータ (Flags)"
    rows={[
      { name: "--json", type: "flag", description: "機械可読な JSON レポートを出力します。" },
      { name: "--no-cache", type: "flag", description: "ローカルキャッシュをバイパスし、最新情報を npm に直接クエリします。" },
    ]}
  />

  `doctor` コマンドの終了コードは常に `0` です。自動化プロセスは、JSON 出力の `report.stale` または `report.coreUpdateAvailable` を解析して実行ロジックを決定する必要があります。
</ApiEntry>

<ApiEntry
  name="upgrade"
  kind="command"
  importFrom="open-press upgrade ."
  summary="ワークスペースの依存関係とエージェントスキルをアップグレードします。このコマンドはプロジェクトファイルを変更します。"

</ApiEntry>
