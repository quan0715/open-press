このドキュメントでは、OpenPress にさまざまな Skills をインストールして利用する方法を説明します。

## ステップ 1: 利用可能な Skills を理解する

OpenPress は、さまざまなワークフローのニーズを満たすために、いくつかの種類の Skills を提供しています:

- **システム操作:** `openpress` (CLI の使用、検証、エクスポート、ルーティング)、`openpress-upgrade` (パッケージアップグレードと Migration QA)、`openpress-deploy` (デプロイ)。
- **ドキュメント作成:** `openpress-create-pages` (A4 などのページドキュメント)、`openpress-create-slide` (プレゼンテーションドキュメント)。
- **レビュー:** `/apply-comments` (インラインコメントの処理)。
- **外部スターター:** 具体的な例や再利用可能なスターターファイルが必要な場合は、別の starter-bearing skill をインストールします。

## ステップ 2: 外部またはクリエイティブな Skills をインストールする

既存 workspace の組み込み OpenPress skills を更新するには、次を実行します:

```bash
npx open-press skills:sync
```

特別なデザインやテンプレート（ソーシャルカードなど）が必要な場合は、npx を介して外部の skill をインストールできます。例えば、ソーシャルカード skill のインストール:

```bash
npx -y skills@latest add quan0715/openpress-social-card-skill
```

## ステップ 3: Agent を通じて Skill を適用する

インストールが完了したら、Agent を誘導してその skill を使用させます:

1. Agent にその skill の `SKILL.md` の説明を読むように指示します。
2. Agent に skill が提供する `starter/` のサンプルファイルを OpenPress Workspace にコピーまたは適応させます。
3. ページやプレゼンテーション作成の skill の場合は、公式の説明に基づいて対応する `press.tsx` とディレクトリ構造を Agent に生成させます。

## ステップ 4: Skill の出力を検証する

構造の作成やコンテンツの導入にどの Skill を使用した場合でも、プロジェクトの健全性を確認するために、終了する前に必ずビルドを実行してください:

```bash
npm run build
```
