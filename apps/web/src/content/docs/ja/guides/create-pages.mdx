## ステップ 1：環境の確認

開始する前に、環境に Node.js 20 以降、および npm と npx がインストールされていることを確認してください。

## ステップ 2：Workspace の初期化（新規プロジェクトのみ）

まったく新しいプロジェクト環境を作成する場合は、まず以下を実行します：

```bash
npm create @open-press@latest <プロジェクト名>
```

既存のプロジェクトにドキュメントを追加する場合は、このステップをスキップしてください。

## ステップ 3：Skill を使用してページ構造を生成する

Agent を呼び出すか、`openpress-create-pages` ワークフローを実行します。構造を生成するために以下の情報を提供してください：
- アーティファクトの種類（例：レポート、提案書）
- ターゲットオーディエンスとタイトル
- ページサイズ（例：`a4`）

この操作により、MDX ソース、ページコンポーネント、および最初のトピックファイルを含む Press Tree が作成されます。

たとえば、生成された Press 構造は次のようになります：

```tsx
<Press
  slug="proposal"
  title="Investor Proposal"
  page="a4"
  sources={[
    mdxSource({ id: "proposal", preset: "section-folders", root: "proposal/chapters" }),
  ]}
>
  <Cover />
  <Toc source="proposal" maxLevel={2} />
  <Sections source="proposal" />
  <BackCover />
</Press>
```

## ステップ 4：構造を検証する

構造の生成が完了したら、ビルドコマンドを実行して、すべてのファイルと設定が正しいことを確認します：

```bash
npm run build
```

注意：プレゼンテーションを作成する必要がある場合は、`openpress-create-slide` の使用を参照してください。デプロイが必要な場合は、確認後に `openpress-deploy` を使用してください。
