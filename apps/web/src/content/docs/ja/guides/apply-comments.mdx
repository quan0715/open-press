## ステップ 1：保留中のマーカーを見つける

grep または ripgrep を使用して、プロジェクト内のすべてのコメントマーカーを見つけます：

```bash
rg "@openpress-comment" press -n
```

## ステップ 2：コメントの内容とソースコードを確認する

マーカーを含むソースコードファイルを開きます。マーカーの周囲の行を読み、コメントのコンテキストと要件を理解します。

## ステップ 3：変更を適用する

コメントの指示に従い、そのコメントを満たす最小限のソースコードの変更を行います。
- 既存のローカルスタイル、コンポーネント API、および MDX 構造を維持します。
- コメントの意図が明確でない場合は、マーカーを残し、推測で変更を行わないでください。

## ステップ 4：マーカーを削除する

**変更が完了したことを確認した後のみ**、ソースコードから `{@openpress-comment ...}` マーカーの行を削除します。何も変更を行わずにマーカーを直接削除しないでください。

## ステップ 5：変更を検証する

ビルドコマンドを実行して、構造が壊れたり構文エラーが発生したりしていないことを確認します：

```bash
npm run build
```

## ステップ 6：結果を報告する

処理が完了したマーカー ID を記録し、意味が不明確で処理されなかったコメントがあれば報告します。
