`@open-press/cli` 提供三層命令列介面，負責管理 workspace 啟動、建置流程與專案維護。

<div class="callout">
  <strong>介面穩定性保證：</strong> 第一層與第二層指令屬於 1.0 穩定契約規範。第三層工具專供系統、agents 及 workbench 使用。
</div>

## CLI 層級架構

<div class="cli-grid">
  <a class="cli-card" href="/docs/concepts/cli-lifecycle">
    <p class="cli-card__eyebrow">第一層</p>
    ### 生命週期 (Lifecycle)
    <p>涵蓋日常建置操作。包含：<code>create</code>、<code>dev</code>、<code>build</code>、<code>preview</code>、<code>typecheck</code>。</p>
  </a>

  <a class="cli-card" href="/docs/reference/cli-outputs">
    <p class="cli-card__eyebrow">第二層</p>
    ### 輸出目標 (Output targets)
    <p>管理標準 HTML 以外之交付物產出。包含：<code>openpress:pdf</code>、<code>openpress:image</code> 與 <code>openpress:deploy</code>。</p>
  </a>

  <a class="cli-card" href="/docs/reference/cli-tools">
    <p class="cli-card__eyebrow">第三層</p>
    ### 工具 (Tools)
    <p>系統維護與代理工具。包含：<code>search</code>、<code>replace</code>、<code>inspect</code>、<code>doctor</code>、<code>upgrade</code> 與 <code>skills:sync</code>。</p>
  </a>
</div>

## API 參考格式

本手冊中每個 CLI 指令皆採用 API 條目格式記錄：

- **基本資訊**：包含指令名稱、種類標籤、調用語法及單行摘要。
- **參數列表**：條列所有支援的旗標 (Flags) 及屬性。
- **範例**：標準使用情境示範。

調用語法會依據情境顯示原生的 `open-press` 指令或 `npm run` 封裝別名。
