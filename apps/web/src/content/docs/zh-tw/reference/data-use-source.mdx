    <h3>範例：讀取已註冊的來源</h3>

```tsx
import { useSource } from "@open-press/core";

function CustomTableOfContents() {
  const story = useSource("story");
  return (
    <ul>
      {story.sections.map((s) => (
        <li key={s.id}>{s.title}</li>
      ))}
    </ul>
  );
}
```

    <h3>範例：帶有型別參數的讀取</h3>

```tsx
import { useSource } from "@open-press/core";
import type { ResolvedSource } from "@open-press/core";

interface StorySource extends ResolvedSource {
  sections: Array<{ id: string; title: string; chapterCount: number }>;
}

function ChapterCounts() {
  const story = useSource<StorySource>("story");
  return story.sections.map((s) => (
    <div key={s.id}>{s.title}: {s.chapterCount} chapters</div>
  ));
}
```

  <h2>何時該使用它</h2>

  <ul>
    <li>
      <strong>客製化的 frame 輔助工具</strong> — 當您正在編寫您自己的 <code>&lt;Sections&gt;</code> 替代品，或是為文稿輔助工具編寫包裝器 (wrapper) 時，您會需要透過這個方式來讀取來源。
    </li>
    <li>
      <strong>客製化目錄 (TOC)</strong> — 當內建的 <code>&lt;Toc&gt;</code> 不符合需求，而您需要從頭渲染章節標題 / 頁碼時。
    </li>
    <li>
      <strong>由資料驅動的介面 (chrome)</strong> — 當頁首、頁尾或是封面需要從來源註冊 (而不是從設定檔) 讀取作者 / 版本 / 章節數量時。
    </li>
  </ul>

  <h2>何時不該使用它</h2>

  <ul>
    <li>
      <strong>在 MDX 散文內文裡面</strong> — MDX 區塊已經在來源 context 中進行渲染；請直接使用區塊層級的 props，而不是重新透過 hook 讀取。
    </li>
    <li>
      <strong>在 <code>&lt;Press&gt;</code> 外部</strong> — hook 會拋出錯誤。如果您在文件樹狀結構外部需要來源資料，請從擁有該來源的 <code>press/&lt;slug&gt;/press.tsx</code> 模組或共用的來源模組中讀取。
    </li>
  </ul>
