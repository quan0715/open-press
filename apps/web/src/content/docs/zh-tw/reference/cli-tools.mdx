第三層 CLI 工具支援 workspace 操作、結構檢查與狀態更新。這些工具獨立於日常建置生命週期。

## 原始碼操作

<ApiEntry
  name="search"
  kind="command"
  importFrom="open-press search . <query> [--json]"
  summary="於 MDX 來源執行全文搜尋。回傳符合項目的檔案路徑、行號、欄位與預覽字串。"
>
  <PropsTable
    title="參數 (Flags)"
    rows={[
      { name: "<query>", type: "string", required: true, description: "搜尋目標。支援純文字或正規表示式。" },
      { name: "--json", type: "flag", description: "輸出機器可讀之 JSON 格式。" },
      { name: "--scope", type: "string", default: '"content"', description: "指定搜尋範圍。`content` (僅 MDX 內文) 或 `all` (包含 frontmatter 與 metadata)。" },
      { name: "--case-sensitive", type: "flag", description: "強制區分大小寫。預設為不區分大小寫。" },
    ]}
  />

  ### 範例：JSON 搜尋

  ```bash
  open-press search . "Figure" --json | jq '.matches[] | {file: .path, line, preview}'
  ```
</ApiEntry>

<ApiEntry
  name="replace"
  kind="command"
  importFrom="open-press replace . <from> <to> [--apply]"
  summary="於 MDX 來源執行批次字串替換操作。預設模式為唯讀預覽。"
>
  <PropsTable
    title="參數 (Flags)"
    rows={[
      { name: "<from>", type: "string", required: true, description: "搜尋目標字串或正規表示式。" },
      { name: "<to>", type: "string", required: true, description: "替換用字串。" },
      { name: "--apply", type: "flag", description: "套用變更並寫入檔案。未提供此旗標時僅輸出預覽。" },
      { name: "--scope", type: "string", default: '"content"', description: "同 `search` 指令之範圍設定。" },
      { name: "--include-code", type: "flag", description: "擴大替換範圍至程式碼區塊 (fenced code blocks)。預設略過以防誤改程式碼。" },
      { name: "--case-sensitive", type: "flag", description: "強制區分大小寫。" },
      { name: "--json", type: "flag", description: "輸出機器可讀之 JSON 報告。" },
    ]}
  />

  ### 範例：執行替換

  ```bash
  open-press replace . "舊片語" "新片語" --apply
  ```
</ApiEntry>

## 環境與狀態管理

<ApiEntry
  name="doctor"
  kind="command"
  importFrom="open-press doctor ."
  summary="回報 workspace 相依性與套件版本狀態。檢查 framework 版本與 agents skills。結果快取 24 小時。"
>
  <PropsTable
    title="參數 (Flags)"
    rows={[
      { name: "--json", type: "flag", description: "輸出機器可讀之 JSON 報告。" },
      { name: "--no-cache", type: "flag", description: "繞過本機快取，直接向 npm 查詢最新資訊。" },
    ]}
  />

  `doctor` 指令退出碼恆為 `0`。自動化流程應解析 JSON 輸出之 `report.stale` 或 `report.coreUpdateAvailable` 判斷執行邏輯。
</ApiEntry>

<ApiEntry
  name="upgrade"
  kind="command"
  importFrom="open-press upgrade ."
  summary="升級 workspace 依賴與 agents skills。source migration 與 QA 請使用 `openpress-upgrade` skill。"
>
  請與 `doctor` 配合使用，在執行前確保已建立 Git 版本控制快照。此指令會刷新套件與 skills；workspace source migration 由 agent 負責：使用 `openpress-upgrade` 讀取適用的 migration docs、掃描 `press/`、套用已確認的 source edits，並循環執行 Migration QA checkpoints。
</ApiEntry>
