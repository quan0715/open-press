此文件描述 OpenPress 第二層 CLI 指令，負責產生 HTML 套件包以外的輸出格式（PDF、PNG 圖片）與部署產物。這些指令包裝了依賴項（Chromium、轉接器 SDK）以執行特定任務，獨立於基礎的 `build` 指令。

<ApiEntry
  name="openpress:deploy"
  kind="command"
  importFrom="npm run openpress:deploy"
  summary="根據指定的轉接器 (Adapter) 執行部署流程。"
>
  此指令依序執行建置、產生 PDF 階段產物、將部署 metadata 寫入 `deploy.json`，並調用配置的部署轉接器。

  <PropsTable
    title="參數 (Flags)"
    rows={[
      {
        name: "--confirm",
        type: "flag",
        description: "實際部署發佈之必要參數。繞過互動式提示，適用於 CI 環境。",
      },
      { name: "--dry-run", type: "flag", description: "執行飛行前檢查並印出指令序列。不實際進行發佈與上傳。" },
    ]}
  />

  ### 範例：部署操作

  ```bash
  # 預覽部署流程 (Dry run)
  npm run openpress:deploy:dry-run

  # 實際執行部署 (含確認旗標)
  npm run openpress:deploy -- --confirm
  ```
</ApiEntry>

## 部署轉接器 (Adapters)

作用中的轉接器由 `package.json` 中的 `"openpress.deploy.adapter"` 屬性決定。

<ApiEntry
  name='"cloudflare-pages"'
  kind="config"
  importFrom={`// package.json\n"openpress": {\n  "deploy": { "adapter": "cloudflare-pages", "projectName": "...", "source": ".deploy" }\n}`}
  summary="調用 npx wrangler pages deploy。必要屬性：projectName。選擇性屬性：commitDirty。"
/>

Wrangler 環境必須在執行部署命令的機器上預先完成驗證。對於未提供專屬轉接器的主機環境，請直接使用標準的靜態建置輸出。
