## 步驟 1：尋找待處理的標記

使用 grep 或 ripgrep 來尋找專案中的所有註解標記：

```bash
rg "@openpress-comment" press -n
```

## 步驟 2：檢查註解內容與原始碼

開啟包含標記的原始碼檔案。閱讀該標記周圍的行數，了解註解的上下文與需求。

## 步驟 3：套用修改

根據註解的指示，進行能滿足該註解的最小原始碼變更。
- 保留原有的區域樣式、元件 API 與 MDX 結構。
- 如果註解的意思不夠明確，請將標記保留，不要進行猜測修改。

## 步驟 4：移除標記

**只有在確認修改已完成後**，才將該行 `{@openpress-comment ...}` 標記從原始碼中刪除。請勿在沒有進行任何修改的情況下直接移除標記。

## 步驟 5：驗證變更

執行建置指令以確保沒有破壞任何結構或產生語法錯誤：

```bash
npm run build
```

## 步驟 6：回報結果

將處理完畢的標記 ID 記錄下來，並回報任何因為語意不清而未處理的註解。
