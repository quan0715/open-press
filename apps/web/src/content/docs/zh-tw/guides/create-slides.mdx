  <h2>工作流程</h2>

  <ol>
    <li>檢查 Node、npm 與 npx。OpenPress 需要 Node.js 20 或更新版本；框架開發與 Cloudflare Pages build 建議使用 Node.js 24。</li>
    <li>偵測是全新 workspace 還是已存在的 <code>&lt;Workspace&gt;</code>。</li>
    <li>收集主題、受眾、標題、投影片數量、密度、動畫、視覺方向、資產，以及 slug。</li>
    <li>建立一個簡報 Press，預設使用 <code>type="slides"</code> 與 <code>page="slide-16-9"</code>。</li>
    <li>生成 <code>DeckSlide</code>、<code>layouts/</code> 以及 <code>ui/</code>。</li>
    <li>透過 protocol compound components 與共享 Tailwind <code>op-*</code> slide layer 進行樣式設定；只有在必要時才把 local CSS 當 escape hatch。</li>
    <li>簡報結構就位後執行 <code>npm run build</code>。</li>
  </ol>

  <h2>預設形狀</h2>

  <h3>範例：簡報 Press</h3>

```tsx
<Press slug="slide" title="Deck Title" type="slides" page="slide-16-9">
  <TitleSlide id="cover" title="Deck Title">
    <TitleSlide.Title objectId="title">Deck Title</TitleSlide.Title>
    <TitleSlide.Description objectId="description">
      One-line audience promise.
    </TitleSlide.Description>
  </TitleSlide>

  <TitledContentSlide id="problem-context" title="Problem Context">
    <TitledContentSlide.Eyebrow objectId="eyebrow">Context</TitledContentSlide.Eyebrow>
    <TitledContentSlide.Title objectId="title">Problem Context</TitledContentSlide.Title>
    <TitledContentSlide.Content>
      <Text as="p" objectId="summary">Write visible slide content directly in JSX.</Text>
    </TitledContentSlide.Content>
  </TitledContentSlide>
</Press>
```

  <h3>範例：DeckSlide</h3>

```tsx
export function DeckSlide({ id, variant, title, children }) {
  return (
    <Slide
      id={id}
      title={title}
      className="op-slide-page"
    >
      <div className="op-slide-shell" data-variant={variant}>
        <main className="op-slide-main">{children}</main>
        <footer className="op-slide-chrome-footer">
          <PageFolio currentFormat="plain" />
        </footer>
      </div>
    </Slide>
  );
}
```

  <h2>版面配置與 UI 邊界 (Boundary)</h2>

  <ul>
    <li><code>components/</code> 存放簡報層級的包裝元件，例如 <code>DeckSlide</code>。</li>
    <li><code>layouts/</code> 存放全螢幕的版面配置，例如 <code>TitledContentSlide</code> 與 <code>ChapterOpenerSlide</code>。</li>
    <li><code>ui/</code> 存放可重複使用的內容元素，例如 <code>Timeline</code>、<code>KpiCard</code> 與 <code>ImageFrame</code>。</li>
    <li>基於文字的複合插槽應將 <code>TextProps</code> 傳遞給 <code>Text</code>，並在可見的文字上使用 <code>objectId</code> 以支援行內編輯的原始碼對應。</li>
    <li>預設輸出應該在版面配置插槽內部行內組合可見內容，而不是將每一張投影片隱藏在一次性元件背後。</li>
  </ul>

  <h2>頁碼 (Page folio)</h2>

  <p>
    投影片版面配置應使用 <code>PageFolio</code> 而非硬寫死的頁碼。
    匯出管線會解析最終的 frame 順序，因此相同的元件可以在閱讀器、
    PDF 以及圖片匯出中運作。
  </p>

  <h3>範例：Folio 格式</h3>

```tsx
<PageFolio currentFormat="2-digit" />
// 01

<PageFolio variant="slash" currentFormat="2-digit" totalFormat="2-digit" />
// 04 / 35

<PageFolio variant="prefix" prefix="p " />
// p 4
```

  <h2>邊界 (Boundary)</h2>

  <ul>
    <li>不建立報告、書籍、提案或其他以頁面為基礎的產物。請使用 <code>openpress-create-pages</code>。</li>
    <li>不透過 create 執行升級或遷移。請使用 <code>openpress</code>。</li>
    <li>不部署。在明確獲得使用者確認後請使用 <code>openpress-deploy</code>。</li>
  </ul>
