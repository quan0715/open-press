<div class="callout">
    <strong>邊界。</strong> OpenPress 不負責抓取 starter、解析 skill 版面配置，或是運作樣板市集。Agents 安裝或讀取 skills，然後將 skill 所擁有的 starter/範例複製或改編到一個 OpenPress Workspace 中。
  </div>

  <h2>OpenPress 營運 skills</h2>

  <div class="skills-grid">
    <article class="skills-card">
      <p class="skills-card__eyebrow">系統 (System)</p>
      <h3>openpress</h3>
      <p>共用的運作契約：CLI 使用方式、原始碼/產生器邊界、驗證、匯出、升級，以及擁有者路由。</p>
    </article>

    <a class="skills-card" href="/docs/guides/create-pages">
      <p class="skills-card__eyebrow">建立 (Create)</p>
      <h3>openpress-create-pages</h3>
      <p>建立以頁面為基礎的產物：必要時的 workspace bootstrap、MDX 原始碼版面配置、頁面元件與首個主題。</p>
    </a>

    <a class="skills-card" href="/docs/guides/create-slides">
      <p class="skills-card__eyebrow">建立 (Create)</p>
      <h3>openpress-create-slide</h3>
      <p>建立簡報：投影片 Press Tree、DeckSlide、投影片版面配置、可重複使用的 UI 元素、簡報結構、資產與首個主題。</p>
    </a>

    <a class="skills-card" href="/docs/guides/apply-comments">
      <p class="skills-card__eyebrow">審閱 (Review)</p>
      <h3>/apply-comments</h3>
      <p>遍歷待處理的 <code>@openpress-comment</code> 標記、套用編輯、移除標記。</p>
    </a>

    <article class="skills-card">
      <p class="skills-card__eyebrow">部署 (Deploy)</p>
      <h3>openpress-deploy</h3>
      <p>準備部署設定、執行 dry-run，並且只有在明確確認目標專案名稱後才發佈。</p>
    </article>
  </div>

  <h2>包含 Starter 的 skills</h2>

  <p>
    包含 Starter 的 skills 依然是正常的 skills。它們的 <code>starter/</code> 檔案是供 agents 檢查、複製和改編的素材。它們不修改 OpenPress 的執行期行為。
  </p>

  <div class="skills-grid">
    <article class="skills-card">
      <p class="skills-card__eyebrow">長文 (Long form)</p>
      <h3>editorial-monograph</h3>
      <p>A4 提案、報告、白皮書、產品規格與長篇編輯文件。</p>
    </article>

    <article class="skills-card">
      <p class="skills-card__eyebrow">工作文件 (Working document)</p>
      <h3>claude-document</h3>
      <p>具溫度的 A4 筆記、簡報、規格、研究摘要與學習材料。</p>
    </article>

    <article class="skills-card">
      <p class="skills-card__eyebrow">學術 (Academic)</p>
      <h3>academic-paper</h3>
      <p>研究論文、研討會風格文章、摘要、參考文獻與編號章節。</p>
    </article>
  </div>

  <h2>外部創意 skills</h2>

  <p>
    外部創意 skills 擁有品味、接收、視覺配方、影像策略、範例與起始檔案。OpenPress 則擁有固定的頁面執行期、可見的 workspace、原始碼管理、渲染、驗證與輸出命令。
  </p>

  <div class="skills-grid">
    <article class="skills-card skills-card--wide">
      <p class="skills-card__eyebrow">社交卡片 (Social card)</p>
      <h3>openpress-social-card-skill</h3>
      <p>
        透過 <code>npx -y skills@latest add quan0715/openpress-social-card-skill</code> 安裝。
        該 skill 會引導 agent 安裝 OpenPress 套件，然後將自己的社交卡片 starter 複製或改編到 workspace 中。
      </p>
    </article>
  </div>
