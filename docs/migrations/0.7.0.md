## Summary

v0.7.0 replaces the book-shaped React exporter with the **Press Tree
Architecture** — a small, generic render kernel. `document/index.tsx` now
default-exports a `<Press>` component instead of named `cover` / `toc` /
`backCover` shell exports. Sources are declared once via `mdxSource()`;
section-flow pagination, TOC pages, and chapter numbering are produced at
build time by the new pipeline. Three primitives ship in `@open-press/core`
(`Press`, `Frame`, `MdxArea`); long-form helpers live in
`@open-press/core/manuscript`; MDX source descriptors in
`@open-press/core/mdx`.

This is a **clean break** — no v0.6 compatibility layer. Workspaces must
migrate `document/index.tsx`. Pre-1.0 minor bump is appropriate (no production
deployments to break).

## Document-level changes

### `document/index.tsx` shell exports → default Press component

- **Find**: `rg '^export const (cover|toc|backCover)' document/index.tsx`
- **Replace**: rewrite to a `<Press>` default export. Each old shell becomes a
  workspace-owned `<Frame>`-returning component:

```tsx
import { Frame, Press } from "@open-press/core";
import { mdxSource } from "@open-press/core/mdx";
import { Sections, Toc } from "@open-press/core/manuscript";
import Page from "./components/Page";

export const config = { title: "…", sourceDir: "chapters" };

export const sources = {
  story: mdxSource({ preset: "section-folders", root: "chapters" }),
};

function Cover() {
  return (
    <Frame frameKey="cover" role="manuscript.cover" chrome={false}>
      …existing cover content…
    </Frame>
  );
}

function BackCover() {
  return (
    <Frame frameKey="back-cover" role="manuscript.back-cover" chrome={false}>
      …existing back-cover content…
    </Frame>
  );
}

export default function MyPress() {
  return (
    <Press>
      <Cover />
      <Toc source="story" />
      <Sections source="story" page={Page} />
      <BackCover />
    </Press>
  );
}
```

- **Why**: Core no longer knows about `cover` / `toc` / `chapter` / `back-cover`
  as built-in concepts. The Press tree makes the document shape explicit.

### `document/components/Page.tsx` — `BaseContentPage` → `<Frame>` + `<MdxArea>`

- **Find**: `rg 'BaseContentPage|children:\s*ReactNode' document/components/`
- **Replace**: workspace-owned `Page` now receives `SectionsPageProps` and
  declares a slot via `<MdxArea>`:

```tsx
import { Frame, MdxArea } from "@open-press/core";
import type { SectionsPageProps } from "@open-press/core/manuscript";

export default function Page({
  frameKey, chainId, pageIndex, totalPages, sectionSlug, sectionTitle, sectionTone,
}: SectionsPageProps) {
  return (
    <Frame frameKey={frameKey} role="manuscript.content"
      data-page-index={pageIndex} data-total-pages={totalPages}
      data-section-id={sectionSlug} data-chapter-tone={sectionTone}>
      <div className="page-frame">
        <header className="page-header" aria-hidden="true" />
        <main className="page-body">
          <MdxArea chainId={chainId} />
        </main>
        <footer className="page-footer">
          <span className="footer-left">{sectionTitle}</span>
          <span className="footer-right">{pageIndex + 1}</span>
        </footer>
      </div>
    </Frame>
  );
}
```

- **Why**: `BasePage`, `BaseCoverPage`, `BaseTocPage`, `BaseContentPage`,
  `BaseBackCoverPage` are removed. The new primitives are `Frame` (page
  surface) and `MdxArea` (measurable content slot).

### Section-scoped CSS attribute: `data-chapter-slug` → `data-section-id`

- **Find**: `rg 'data-chapter-slug' document/`
- **Replace**: switch to `data-section-id` on the workspace `Page` (the
  example above already does this) and update any per-section CSS selectors
  in `document/chapters/<slug>/styles/*.css`:

  ```css
  /* before */ [data-chapter-slug="intro"] :where(h2) { … }
  /* after  */ [data-section-id="intro"]   :where(h2) { … }
  ```

  The engine's CSS scoper now emits `[data-section-id]` automatically; the
  selector rewrite is only needed if you wrote raw selectors in custom CSS.

- **Why**: Core vocabulary uses `Section`; `Chapter` is alias-only.

- (none — the bundled `openpress` SKILL is updated in this release but no

## CLI changes

- **Removed**: `npx open-press migrate-to-react`. The v0.6 → v0.7 jump is
  large enough that no auto-migration is provided. Workspaces should follow
  the document-level steps above by hand or with agent help.
- All other commands (`init`, `validate`, `export`, `render`, `pdf`, `dev`,
  `preview`, `deploy`, `doctor`, `upgrade`, `search`, `replace`, `inspect`)
  are unchanged in surface.

## Runtime / API changes

For users with custom `document/components/` that import from
`@open-press/core`:

- **Removed**: `BasePage`, `BaseCoverPage`, `BaseTocPage`, `BaseContentPage`,
  `BaseBackCoverPage`, `PageProps`, `BasePageProps`, `BaseContentPageProps`,
  `BaseShellPageProps`, `PageKind` types.
- **Added**: `Press`, `Frame`, `MdxArea`, `useSource`. Types:
  `FrameProps`, `MdxAreaProps`, `MdxAreaOverflow`, `PressContextValue`,
  `ResolvedSource`, `SourceNode`, `OutlineItem`, `FrameAllocation`.
- **Kept**: `BaseFigure`, `BaseCallout`, `BaseFigureProps`, `BaseCalloutProps`
  (content primitives — unchanged).
- **New subpaths**:
  - `@open-press/core/mdx` exports `mdxSource()` plus descriptor types.
  - `@open-press/core/manuscript` exports `Toc`, `Sections`, `Chapters`
    (alias for `Sections`), and `SectionsPageProps` / `SectionsOpenerProps`.

## Reader / document.json schema

- `meta.version` is now `"openpress-press-tree-v1"` (was
  `"openpress-react-export-v1"`).
- `source.frames` and `source.chains` are new in `document.json`; the
  workbench / public reader consume them automatically.
- The runtime DOM re-pagination (`src/openpress/pagination.ts`) is removed —
  build-time pagination is the single source of truth, including TOC overflow
  pages. The reader simply renders pre-paginated frames now.

## Manual steps

1. Rewrite `document/index.tsx` to default-export a `<Press>` component as
   shown above. The agent can do this automatically with user confirmation.
2. Rewrite `document/components/Page.tsx` (and any chapter-opener / back-cover
   component) to use `<Frame>` + `<MdxArea>` instead of `BaseContentPage`.
3. Update raw CSS selectors that used `data-chapter-slug` (rare — most
   workspaces keep section scoping in chapter-css files which the engine
   rewrites automatically).
4. Run `npm run openpress:validate && npm run openpress:render` to confirm
   the new pipeline produces the expected page count and TOC entries.
5. Re-run `npx skills upgrade` to refresh the `.agents/skills/` directory.
